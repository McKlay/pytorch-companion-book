    Chapter 4: torch.Tensor¶

    ‚ÄúThe universe of PyTorch begins with a single tensor‚Äîand everything else builds on top of it.‚Äù

    4.1 Tensor Creation Methods¶

    Let‚Äôs explore how to instantiate tensors like a boss. PyTorch provides multiple ways depending on your use case:

    ‚ñ∂ Basic Constructor¶

    torch.tensor([1, 2, 3])                 # From Python list
torch.tensor([[1.0, 2.0], [3.0, 4.0]])  # 2D Float Tensor

    ‚ñ∂ Pre-filled Factories¶

    torch.zeros(2, 3)           # All zeros
torch.ones(2, 3)            # All ones
torch.full((2, 2), 42)      # All elements are 42
torch.eye(3)                # Identity matrix
torch.arange(0, 10, 2)      # Like Python‚Äôs range()
torch.linspace(0, 1, 5)     # 5 values between 0 and 1

    ‚ñ∂ Like Another Tensor¶

    x = torch.ones(2, 2)
torch.zeros_like(x)
torch.rand_like(x)

    üìê 4.2 Tensor Properties¶

    Every tensor has a few critical attributes:
    üî∏ shape and size() 

    x.shape            # torch.Size([2, 3])
x.size()           # Same as above
    üî∏ dtype ‚Äì data type 
    x.dtype            # e.g., torch.float32
x = x.to(torch.int64)  # change type
    üî∏ device ‚Äì CPU or CUDA 
    x.device           # Shows current device
x = x.to('cuda')   # Move to GPU
    üî∏ requires_grad 
    x = torch.tensor([1.0, 2.0], requires_grad=True)
x.requires_grad    # True

    4.3 Tensor Operations¶

    PyTorch supports extensive elementwise operations directly on tensors:

    Arithmetic 

    x + y
x - y
x * y
x / y
x ** 2
    Comparison 
    x == y
x > y
x != y
    Logical 
    torch.logical_and(x > 0, x < 1)

    ‚úÖ These are vectorized ‚Äî no need for loops!

    4.4 Reshaping & Reorganizing Tensors¶

    These tools let you morph tensor shapes without changing their content.

    Reshape & View 

    x.view(-1)           # Flatten (requires contiguous memory)
x.reshape(2, 3)      # Flexible reshape
    Squeeze & Unsqueeze 
    x = torch.zeros(1, 3, 1)
x.squeeze()          # Remove dim=1 ‚Üí (3,)
x.unsqueeze(0)       # Add dim at index 0 ‚Üí (1, 3, 1)
    Permute & Transpose 
    x = torch.randn(2, 3, 4)
x.permute(2, 0, 1)   # Reorders dimensions
x.transpose(0, 1)    # Swaps two dims only

    üîÅ Use permute() for high-dimensional tensors (images, etc.)

    4.5 Indexing & Slicing¶

    Basic and advanced ways to access tensor values: 

    x[0]             # First row
x[:, 1]          # Second column
x[1:, :]         # All rows except the first
    Also supports: 
    x[x > 0]                         # Boolean mask
x[torch.tensor([0, 2])]          # Indexing with tensor

    These are the same ideas as NumPy ‚Äî but with GPU support.

    4.6 Autograd Compatibility¶

    One of PyTorch‚Äôs killer features is automatic differentiation ‚Äî made possible because every Tensor can carry its computation history. 

    x = torch.tensor([2.0, 3.0], requires_grad=True)
y = x ** 2 + 3
z = y.sum()
z.backward()
x.grad  # ‚àÇz/‚àÇx
    Don‚Äôt track: 
    with torch.no_grad():
    result = model(x)

    üî• If requires_grad=True, the tensor is part of the computation graph. Perfect for training neural nets.

    4.7 Miscellaneous API Tricks¶

    Clone vs Detach: 

    x.clone()        # Returns a copy
x.detach()       # Returns a tensor with no grad-tracking
    Checking if a tensor is contiguous: 
    x.is_contiguous()

    Useful when using .view() which demands contiguous memory layout.

    4.8 Summary¶

    • torch.Tensor is more than just an array ‚Äî it has memory, gradient, and device awareness.

    • You can create tensors using many constructors (zeros, arange, full, etc.)

    • Tensors support rich operations: math, reshape, slice, compare, move to CUDA.

    • They're also autograd-aware ‚Äî making them perfect for deep learning.
