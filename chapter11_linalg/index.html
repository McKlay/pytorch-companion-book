    Chapter 11: torch.linalg¶

    ‚ÄúWith great matrices comes great responsibility.‚Äù

    11.1 What is torch.linalg?¶

    torch.linalg is PyTorch‚Äôs modern linear algebra module, introduced to offer more stable, consistent, and NumPy-compatible operations compared to older functions like torch.svd() or torch.eig().

    It handles:

    • Matrix decompositions
    • Solvers
    • Norms
    • Eigenvalues
    • Inverses
    • Determinants

    üí° Bonus: most functions support batched operations, autograd, and GPU acceleration.

    11.2 Matrix Inversion and Solving Linear Systems¶

    ‚û§ Invert a matrix¶

    A = torch.randn(3, 3)
A_inv = torch.linalg.inv(A)

    ‚ö†Ô∏è Inverting is expensive. If you're solving Ax = b, use a solver instead.

    ‚û§ Solve linear system Ax = b¶

    A = torch.tensor([[3.0, 1.0], [1.0, 2.0]])
b = torch.tensor([9.0, 8.0])
x = torch.linalg.solve(A, b)

    ‚úÖ Preferred over computing inv(A) @ b ‚Äî more numerically stable.

    11.3 Matrix Determinant and Rank¶

    ‚û§ Determinant¶

    A = torch.tensor([[1.0, 2.0], [3.0, 4.0]])
torch.linalg.det(A)

    ‚û§ Matrix Rank¶

    torch.linalg.matrix_rank(A)

    Useful for checking if a matrix is invertible or has full column rank.

    11.4 Norms and Condition Numbers¶

    ‚û§ Norms¶

    x = torch.tensor([3.0, 4.0])
torch.linalg.norm(x)             # L2 norm

A = torch.tensor([[1.0, 2.0], [3.0, 4.0]])
torch.linalg.norm(A, ord='fro')  # Frobenius norm

    Other ord options: 'nuc' (nuclear), 1, 2, inf, -inf

    ‚û§ Condition Number¶

    torch.linalg.cond(A)

    High condition numbers indicate numerical instability ‚Äî useful for diagnostics.

    11.5 Eigenvalues and Eigenvectors¶

    ‚û§ Eigendecomposition¶

    A = torch.tensor([[5.0, 4.0], [1.0, 2.0]])
eigenvalues, eigenvectors = torch.linalg.eig(A)

    ‚ö†Ô∏è eigenvalues may be complex64 or complex128.

    ‚û§ For symmetric/Hermitian matrices:¶

    eigenvalues = torch.linalg.eigvalsh(A)  # Faster and more stable

    11.6 SVD (Singular Value Decomposition)¶

    Useful in: - PCA - Low-rank approximation - Image compression 

    U, S, Vh = torch.linalg.svd(A, full_matrices=False)
A_reconstructed = U @ torch.diag(S) @ Vh

    11.7 QR and LU Decomposition¶

    ‚û§ QR decomposition¶

    Q, R = torch.linalg.qr(A)

    ‚û§ LU decomposition¶

    LU, pivots = torch.linalg.lu_factor(A)

    11.8 Relationship with NumPy¶

    PyTorch‚Äôs torch.linalg mirrors numpy.linalg in:
    - Function names
    - Shape conventions
    - Numerical semantics

    But PyTorch:
    - ‚úÖ Supports autograd
    - ‚úÖ Runs on GPU
    - ‚úÖ Handles batch operations

    You can almost always port NumPy linear algebra code directly to PyTorch with minimal edits.

    ‚úÖ 11.9 Summary¶

    Operation Function
    Matrix inverse torch.linalg.inv()
    Solve Ax = b torch.linalg.solve()
    Determinant torch.linalg.det()
    Eigenvalues/vectors torch.linalg.eig()
    SVD torch.linalg.svd()
    Norm torch.linalg.norm()
    R / LU torch.linalg.qr(), lu()

    • Use torch.linalg for numerically stable, batched, and autograd-compatible operations.

    • Avoid inv() when you can use solve().

    • Most functions support GPU ‚Äî just move your tensors to cuda.
